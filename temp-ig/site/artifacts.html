Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems conforming to this implementation guide are expected to declare conformance to one or more of the following capability statements.

CapabilityStatement Cliente Cl Core

CapabilityStatement para definir las caracteristicas mínimas de un cliente con todas las interacciones posibles hacía un servidor
CapabilityStatement Servidor Cl Core

CapabilityStatement para definir las caracteristicas mínimas del servidor para ver las interacciones posibles

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

CL Alergias e Intolerancias

Este Perfil ha sido desarrollado para cubrir las necesidades para describir una Alergia o alguna Intolerancia definida en un paciente.
CL Auditoria

Perfil para generar un recurso que permita desarrollar auditoría de las transacciones realizadas
CL Condición-Diagnóstico

Condición o Diagnósticos de Pacientes
CL Documento

De un formato genérico para documento clínico en Chile.
CL Encuentro Clínico

Definición de encuentro clínico genérico para Chile
CL Inmunización

Definición de un evento de inmunización para las necesidades de interoperabilidad en Chile.
CL Localización

Este perfil determina la ubicación específica para un evento, objeto o Infraestructura específica. Este perfil permite vincular una localización con una organización o espacio físico específico
CL Medicamento

Descripción de un Medicamento
CL Observación

Perfil para definir un mínimo para cualquier observación.
CL Organización

Este Perfil ha sido desarrollado para cubrir las necesidades del Caso de Uso de Receta Electrónica. Sin embargo, se ha modelado para también cubrir las necesidades nacionales de un Recurso de Organización en Salud
CL Paciente

Este Perfil se ha modelado con el fin de cubrir las necesidades nacionales de un Recurso Paciente para un Historial Clínico Nacional
CL Prestador

Este Perfil fue creado para cubrir la descripción de un Prestador a nivel Nacional
CL Provenance

Este Perfil describe la forma de validación respecto a la procedencia de un recurso generado durante un acto clínico.
CL RolClinico

Este Perfil se ha modelado con el fin de cubrir las necesidades la función de Prestador en una organización de Chile, por ejemplo: Hospital, Clínica, Cesfam, farmacias, etc
CL Signos Vitales

Este perfil está basado en el FHIR Vital Signs Profile y definiendo indicaciones adiccionales en el recurso Observation que representa las observaciones de signos vitales. Esto especifica cuales son los elementos principales, extensiones, vocabularios y set de valores que DEBE estar presentes en el recurso y restricciones. Proveendo un piso para estandarizar los desarrollos para casos de uso especificos fomentando la interoperabilidad y su adopción.
